1. The Foundation (Drivers & CUDA)

NVIDIA Drivers: Ensure you are on a recent branch (535 or 550) that fully supports the Ada Lovelace architecture of the 4090s.

CUDA Toolkit 12.x: You need the full toolkit, not just the runtime. You must have nvcc installed and in your $PATH so CMake can compile our custom Mean Pooling kernel.

2. The Deep Learning Stack (cuDNN & TensorRT)

NVIDIA cuDNN: TensorRT relies heavily on cuDNN for optimized math primitives. Ensure a compatible version (usually 8.9+ for CUDA 12) is installed.

TensorRT 10.x: This is the big one. You need the full TensorRT tarball or .deb package installed.

The trtexec Binary: By default, Ubuntu usually buries this in /usr/src/tensorrt/bin/ or inside the extracted tarball directory. Find it, run chmod +x, and symlink it to /usr/local/bin/trtexec so you can call it from anywhere to bake your 4090 engines.

3. The Build Environment

CMake (>= 3.18): Check cmake --version. Ubuntu's default apt repos sometimes have ancient versions. You need at least 3.18 for the CMAKE_CUDA_ARCHITECTURES flag to work.

Rust & Cargo: Since the tokenizers-cpp library is actually a C++ wrapper around the blazing-fast Rust Hugging Face library, the C++ compiler will silently call cargo under the hood to build the static library during the CMake step. Make sure Rust is installed on the server!
